# P5.2 ŌĆö Orbit anchor, pan, and decoupled selection

**Status:** shipped  
**Last updated:** 2026-07-18  
**Roadmap:** [completed ŌĆö P5.2](../roadmap/completed.md)  
**Related:** [P5.1 ŌĆö Selection and view navigation](P5.1-selection-view-nav.md), [P3-rendering](P3-rendering.md), [P4-input-tools](P4-input-tools.md), [P5-factory-ui](P5-factory-ui.md)

## Goal

Decouple **camera framing** from **cell selection** so changing the selected cell no longer recenters the viewport. Introduce an **orbit anchor** that the camera looks at and rotates around, plus explicit **pan** controls and a **recenter** shortcut. Selection-driven slice clipping and LMB pick behavior stay unchanged.

## Motivation

P5.1 ties camera look-at, zoom center, and view rotation to `SelectedCell`. Every arrow-key step or LMB pick shifts the entire view to keep the selection centered. That disrupts focus when the player is surveying a region or stepping selection across a layout they are already viewing.

Players need:

- A **stable viewport** while moving the selection cursor.
- A **separate orbit pivot** for view rotation (Ctrl+arrows) that does not drift with selection.
- **Explicit pan** when they want to look elsewhere without changing selection.
- **Automatic follow** when keyboard navigation pushes selection off-screen, without the jarring full recenter of P5.1.

## Concepts

### Two focal points

P5.1 collapsed camera framing and selection into one resource. P5.2 splits them:

```text
SelectedCell { pos: WorldPos }   // cursor, inspector, slice clip, LMB pick plane
OrbitAnchor  { pos: WorldPos }   // camera look-at, rotation pivot, zoom center
```

| Concern | Source |
|---------|--------|
| Camera look-at | `OrbitAnchor` |
| View rotation pivot (Ctrl+arrows) | `OrbitAnchor` |
| Zoom center | `OrbitAnchor` |
| Inspector / highlight | `SelectedCell` |
| Slice depth / clip / dimming | `SelectedCell` |
| LMB select pick plane | `SelectedCell` depth |
| Arrow / `[`/`]` / wheel | `SelectedCell` only |
| Pan (Shift+arrows, MMB) | `OrbitAnchor` only |
| **`C`** recenter | `OrbitAnchor ŌåÉ SelectedCell` |

**Initialization:** both resources set to the same random non-empty cell from demo seed (same policy as P5.1). Camera frames that cell on first frame.

**Invariant:** both `pos` values are always in `0..WORLD_SIZE` per axis (clamped on set).

### Active slice

The **active slice** is the view plane at the **selected cell's depth** on the current face ŌĆö the same plane used for slice clipping and LMB `pick_slice_cell`:

```text
active_slice_depth = ViewPose::slice_depth(selected.pos)
```

Pan always moves on this plane. After any pan (manual, auto, or `C`), the orbit anchor's depth coordinate on the active face **equals** `active_slice_depth` (anchor stays on the active slice).

### Viewport center cell

Given the current camera transform, orthographic projection, and active slice depth, the **viewport center cell** is the `WorldPos` at the intersection of:

1. The view ray through the viewport center, and  
2. The active slice plane at `active_slice_depth`.

This is the cell that becomes the new `OrbitAnchor.pos` after pan operations and auto-pan. Implementation may use the inverse of `world_hit_to_uv` + `uv_depth_to_cell` ([P5.1 pick math](P5.1-selection-view-nav.md)).

### Visibility test (selection in viewport)

Project `SelectedCell.pos` to **view-plane UV** coordinates (using `ViewPose::u_axis` / `v_axis`). Compare against the visible UV extent derived from camera position, orbit anchor, and `ViewCameraState.zoom_cells`.

The selection is **outside the viewport** when its UV lies beyond the visible half-extents on **either** axis (no margin buffer in v1; tune in implementation notes if edge cells feel sticky).

## Camera framing

```text
look_at  = cell_to_world(orbit_anchor.pos)
camera   = look_at + ViewPose::camera_offset(distance)
distance = WORLD_SIZE * 1.2   // unchanged fixed offset
```

- `apply_camera_framing` runs when **orbit anchor**, **view pose**, or **zoom** changes ŌĆö **not** when only `SelectedCell` changes.
- Orthographic `viewport_height` = `zoom_cells`, centered on the look-at point (orbit anchor).
- Selected cell may appear anywhere on screen, including off-screen.

### Zoom

Unchanged limits (8ŌĆō64 cells on shorter axis). Zoom scales around **orbit anchor**, not selection.

| Input | Action |
|-------|--------|
| Ctrl + mouse wheel | Zoom in/out around orbit anchor |
| Ctrl + `[` / `]` | Zoom out / in |

## Pan

### Manual pan ŌĆö Shift + arrow keys

Each **Shift + arrow** (no Ctrl) pans the view **one quarter of the current viewport** along the corresponding screen direction:

| Key | Pan direction |
|-----|---------------|
| Shift + Up | +V (screen up) |
| Shift + Down | ŌłÆV |
| Shift + Left | ŌłÆU (screen left) |
| Shift + Right | +U |

**Per keypress:**

1. Translate orbit anchor on the active slice UV plane by `┬╝ ├Ś zoom_cells` cells in that direction (quantize to whole-cell steps on the UV axes; clamp anchor UV so the viewport stays within world bounds).
2. Re-apply camera framing.
3. `SelectedCell` is **not** modified.

Shift+arrow and plain arrow are distinguished by Shift (same pattern as palette Shift+digit). Ctrl+arrow remains view rotation. **Shift+Ctrl+arrow** is a distinct unmapped combination ŌĆö no action (not rotate, not pan, not selection).

### Manual pan ŌĆö middle mouse button drag

| Input | Action |
|-------|--------|
| **MMB drag** | Continuous pan on the active slice |

**While MMB held:**

1. Map cursor delta to a translation on the active slice UV plane (scale by `zoom_cells` and viewport pixel size so drag feels 1:1 with content).
2. Move orbit anchor accordingly (continuous; may sub-cell during drag).
3. On **MMB release**, snap orbit anchor to the **viewport center cell** on the active slice (integer `WorldPos`).

`SelectedCell` is not modified.

### Recenter ŌĆö `C`

| Input | Action |
|-------|--------|
| **`C`** | `orbit_anchor.pos = selected.pos`; re-apply camera framing |

Equivalent to P5.1's implicit "frame selection" behavior, now explicit. Does not change `ViewPose` or zoom.

### Auto-pan ŌĆö selection leaves viewport

Triggered after **arrow-key UV navigation** or **LMB drag selection tracking** (P5.1 LMB-drag parity) when the new selection position is **outside the viewport** per the visibility test above.

**Per triggering step** (one arrow press or one drag selection update):

1. Determine the screen direction of the move (the `ScreenDir` of the arrow key, or the sign of UV delta for drag).
2. Pan the view **┬╝ viewport** in that direction (same delta as one Shift+arrow press).
3. Set `orbit_anchor.pos` to the **viewport center cell** on the active slice.

If the selection remains outside after one ┬╝ pan (e.g. zoomed far in), **subsequent** arrow steps in the same off-screen direction repeat auto-pan until the selection re-enters the viewport or world bounds block further pan.

Auto-pan does **not** run for:

- `[` / `]` / wheel depth steps (depth moves are along the view axis; selection stays at same UV)
- LMB click without drag (selection stays at pick UV on the active slice, typically visible)
- RMB tool application

**No auto-pan on depth step** even if selection depth changes relative to camera ŌĆö clip updates, but UV position is unchanged.

## View rotation (unchanged semantics, new pivot)

Ctrl+arrow tumbling is unchanged from P5.1 (`ViewPose::rotate`). The pivot is **`OrbitAnchor`**, not `SelectedCell`:

| Key | Action |
|-----|--------|
| Ctrl + Up/Down/Left/Right | ┬▒90┬░ tumble around orbit anchor |

- Orbit anchor **world position does not change** during rotation.
- Selected cell stays at its world coordinate; it **appears to move on screen** when the view rotates.
- Slice clip is recomputed from selection depth under the **new** face (same as P5.1 depth derivation).
- `T` resets pose to `ViewPose::top_default()`; does not move orbit anchor or selection.

## Slice clip and picking (unchanged)

```text
slice_depth = ViewPose::slice_depth(selected.pos)
```

| Rule | Behavior |
|------|----------|
| Clip plane | Selected cell depth on active face |
| Hidden cells | Between clip plane and camera |
| On-slice highlight | Cells at `slice_depth` |
| Behind-slice dim | Visible but not on slice (`BELOW_SLICE_BRIGHTNESS`) |
| LMB select | `pick_slice_cell(pose, slice_depth, ŌĆ”)` ŌĆö active slice only |
| RMB tools | Unchanged (place at slice depth; erase/inspect on clipped surface) |

To select a cell on a different depth layer, the player still uses `[` / `]` / wheel first ŌĆö same as P5.1.

## Implementation notes

| Decision | Choice |
|----------|--------|
| Pan increment | ┬╝ `zoom_cells` per Shift+arrow and per auto-pan step |
| Anchor depth on pan | Always snap to `active_slice_depth` (selected cell depth) |
| MMB snap | Snap orbit anchor to integer cell on button release |
| Camera framing trigger | Orbit anchor, view pose, zoom ŌĆö not selection-only UV moves |
| Pick math | World-space planes unchanged; camera offset from anchor already handled by ray cast |
| Arrow key modifiers | Plain ŌåÆ move selection; Shift (no Ctrl) ŌåÆ pan; Ctrl (no Shift) ŌåÆ rotate; Shift+Ctrl ŌåÆ no action |
| P6 persistence | Add `OrbitAnchor` to save payload when P6 ships |

### Pan delta (reference)

For a screen-direction unit step on the active slice:

```text
delta_uv = ┬╝ ├Ś zoom_cells ├Ś screen_dir_uv
orbit_anchor_uv += delta_uv   // clamp to world UV bounds
orbit_anchor.depth = active_slice_depth
```

MMB drag uses the same UV axes; scale cursor pixel delta to cell units via viewport size and `zoom_cells`.

### Auto-pan edge case

At world boundary, clamp orbit anchor UV so the viewport does not expose out-of-bounds columns. Selection may remain partially off-screen if it sits on the world edge ŌĆö acceptable for v1.
