# P5.1 — Selection and view navigation

**Status:** shipped  
**Last updated:** 2026-07-17  
**Roadmap:** [completed — P5.1](../roadmap/completed.md)  
**Related:** [next-steps — selection model](../planning/next-steps.md), [P3-rendering](P3-rendering.md), [P4-input-tools](P4-input-tools.md), [P5-factory-ui](P5-factory-ui.md)

## Goal

Replace transient pick-based navigation with a **persistent selected cell** that anchors camera framing, slice depth, zoom, and the inspector. Remap pointer and keyboard bindings so selection is primary (LMB), tools apply on RMB, views rotate via **Ctrl + arrow keys** around the selection, and a **?** help overlay documents all bindings.

## Motivation

Today (P4/P5):

- No persistent selection; `InspectedCell` updates only on RMB / inspect-tool picks.
- Cameras are fixed on world origin; views switch with `1`/`2`/`3` (and `T`/`F`/`L`).
- `ViewSlice` depth is independent per view and decoupled from any cell.
- LMB applies the active tool; RMB inspects.

Players need a stable focal point to orbit orthographic faces, step through depth layers, and edit without losing context. This slice delivers that UX without changing simulation semantics.

## Scope

### In scope

| Area | Summary |
|------|---------|
| **Selected cell** | Always exactly one `WorldPos`; initialized at random from demo world gen |
| **View centering** | Active ortho camera frames the selected cell (pan + zoom relative to selection) |
| **View rotation** | **Ctrl + arrow keys** jump between the six axis-aligned faces around the selection (discrete, not smooth orbit) |
| **Top snap** | `T` returns to **Top** view with default orientation (same as today’s top camera basis) |
| **Slice / depth** | `[` / `]` and mouse wheel step the selected cell along the active view’s depth axis; slice clip derived from selection |
| **Zoom** | **Ctrl + mouse wheel** (and **Ctrl + `[` / `]`**) zoom in/out around the selection; clamped to 8–64 visible cells |
| **Keyboard UV nav** | Arrow keys (no Ctrl) move the selected cell on the active face’s UV plane |
| **Pointer remap** | **LMB** select; **RMB** apply active tool (place / erase); inspect tool mode still available but inspector follows selection |
| **Inspector** | Factory UI inspector always reflects the **selected** cell (live re-read from sim) |
| **Help overlay** | `?` toggles a modal panel listing key and mouse bindings |

### Out of scope

- Multi-select and batch operations ([next-steps](../planning/next-steps.md))
- RMB context menu (tool apply on RMB is the chosen behavior; context menu deferred)
- Smooth 3D camera interpolation between faces
- Configurable keybindings file (P9.2); bindings are hard-coded but documented in help overlay and README
- **ESC** settings menu (reserved; help overlay is the first step toward a global menu)
- Themed world gen or non-demo initial selection policy
- Undo/redo

## Concepts

### Selected cell

```text
SelectedCell { pos: WorldPos }
```

- **Invariant:** `pos` is always valid grid coordinates (`0..WORLD_SIZE` on each axis).
- **Source of truth** for: inspector panel, camera look-at target, depth slice on the active view, keyboard UV navigation start point.
- **Initialization:** `seed_demo_world` (or post-seed hook) picks one **non-empty** cell uniformly at random from the seeded region. If the seeded region is empty (tests), fall back to world center `(250, 250, 250)`.
- **Changes:** LMB pick, arrow keys, `[`/`]`/wheel slice step, and any future selection APIs call a single `set_selection(pos)` that clamps to bounds and triggers camera/slice/inspector refresh.

`InspectedCell` is **superseded** by `SelectedCell` for UI binding. Either remove `InspectedCell` or make it a thin alias updated whenever selection changes (prefer one resource).

### View slice from selection

Depth slice is **derived**, not independent:

```text
slice_depth(view, selected) = selected.{x|y|z} per view depth axis
```

| View | Depth axis | Slice value |
|------|------------|-------------|
| Top / Bottom | Y | `selected.y` |
| Front / Back | Z | `selected.z` |
| Left / Right | X | `selected.x` |

`ViewSlice` resource may remain as a cache updated on selection change, or be replaced by pure functions over `SelectedCell` + `ActiveView`. Pick/surface/clip math continues to use the same `visible_surface(sim, view, slice_depth)` pipeline ([P4](P4-input-tools.md)).

**Slice step** (`[`, `]`, wheel): move `selected` ±1 along the active view’s depth axis (clamped to world bounds). Rebuild render clip when depth changes.

### Six orthographic faces

Extend `OrthoView` from three faces to **six**:

| Face | Camera position (relative to target) | Looks along | Screen +U (horizontal) | Screen +V (vertical) |
|------|--------------------------------------|-------------|--------------------------|------------------------|
| **Top** | +Y | −Y | +X | +Z |
| **Bottom** | −Y | +Y | +X | −Z |
| **Front** | +Z | −Z | +X | +Y |
| **Back** | −Z | +Z | −X | +Y |
| **Left** | −X | +X | +Z | +Y |
| **Right** | +X | −X | −Z | +Y |

Default orientations match the existing Top / Front / Left cameras when the target is world origin. Bottom / Back / Right are the opposite faces with consistent “up” on screen (+Y world up for side views, +Z world north for top/bottom — document in implementation notes if adjusted).

Surface extraction rules generalize P3:

| Face | Visible surface per column |
|------|----------------------------|
| Top | max Y |
| Bottom | min Y |
| Front | max Z |
| Back | min Z |
| Left | min X |
| Right | max X |

Slice clip direction (hide cells between slice and camera) follows P4 per-face convention: **Top/Front/Right** keep `depth ≤ slice`; **Bottom/Back/Left** keep `depth ≥ slice`.

### Camera framing and zoom

```text
ViewCameraState {
  zoom_cells: f32,   // visible extent on the shorter viewport axis, in cell units
}
```

- **Look-at target:** `cell_to_world(selected.pos)`.
- **Pan:** implicit via look-at; no free pan decoupled from selection in this slice.
- **Zoom:** adjust orthographic `viewport_height` (or equivalent) so the shorter screen axis spans `zoom_cells` world units, centered on the selected cell.
- **Default `zoom_cells`:** `32` (comfortable edit zoom on demo region; tune in implementation notes).
- **Limits:**
  - **Zoom in max:** `zoom_cells >= 8` (at least **8 cells** visible on the shorter axis).
  - **Zoom out max:** `zoom_cells <= 64` (avoid sub-pixel cells at default window size).
- **Inputs:** Ctrl + mouse wheel (scroll up = zoom in), Ctrl + `[` / `]` as zoom out / zoom in when Ctrl is held. Without Ctrl, wheel is slice step (see bindings).

Re-center and re-apply zoom when selection or active view changes.

### View rotation (Ctrl + arrows)

Discrete **face adjacency** navigation — “orbit” around the selected cell by jumping to the neighboring orthogonal plane. Arrow keys are interpreted in **screen space** of the current face:

| Key | Action |
|-----|--------|
| **Ctrl + Up** | Rotate to the face above the current view (screen top) |
| **Ctrl + Down** | Rotate to the face below (screen bottom) |
| **Ctrl + Left** | Rotate to the face to the left |
| **Ctrl + Right** | Rotate to the face to the right |

Transition table (screen-up / screen-down / screen-left / screen-right):

| From | Up | Down | Left | Right |
|------|----|------|------|-------|
| Top | Back | Front | Left | Right |
| Bottom | Front | Back | Right | Left |
| Front | Top | Bottom | Left | Right |
| Back | Bottom | Top | Right | Left |
| Left | Top | Bottom | Bottom | Top |
| Right | Top | Bottom | Top | Bottom |

Vertical **Ctrl+Up** cycles Top → Back → Bottom → Front → Top.

Horizontal **Ctrl+Left** cycles Top → Left → Bottom → Right → Top.

(Verify against implementation; table encodes “screen up = world +Z on Top” etc.)

**`T`:** switch to **Top** and reset that camera’s roll to the default Top orientation (does not change selection or zoom).

**Removed:** `1`/`2`/`3` and `F`/`L` as view switches (digit keys `1`–`9` become available for palette without Shift if desired later; **Shift+digit palette unchanged** for this slice).

### UV navigation (arrow keys, no Ctrl)

Move selection on the active face plane by ±1 cell, clamped to world bounds:

| Key | Δ on face UV |
|-----|----------------|
| Up | +V |
| Down | −V |
| Left | −U |
| Right | +U |

(U, V) per face match the table in **Six orthographic faces**. Empty destination cells are allowed — selection is a cursor, not “snap to nearest content.”

### Pointer behavior

| Input | Action |
|-------|--------|
| **LMB** | Ray pick → `set_selection(picked)` if hit resolves to a cell column |
| **LMB drag** | **No tool apply on drag**; optional: update selection to cell under cursor while held (recommended for parity with arrow nav) |
| **RMB** | Apply **active tool** at picked cell (place / erase / inspect per `ToolState`) |
| **RMB drag** | Apply active tool on each newly entered cell (preserves P4 drag-place/erase for RMB) |

**Place** still targets the **current slice depth** (selected cell’s depth on the active axis) for the column under the cursor. **Erase** still targets the **clipped visible surface** cell in that column.

When a tool write changes the cell at the selection coordinate, selection stays on that coordinate and inspector refreshes.

### Inspect

- Inspector panel shows **selected** cell coordinates, type, and state every frame (`refresh` from sim).
- `I` inspect tool and RMB-on-inspect-mode may still log to console; they do not drive a separate inspect target.
- Remove dependency on “last picked inspect cell” in factory UI.

## Default bindings

| Input | Action |
|-------|--------|
| **Arrow keys** | Move selected cell on UV plane |
| **Ctrl + Arrow keys** | Rotate to adjacent orthographic face |
| **`T`** | Snap to Top view (default orientation) |
| **`[`** / **`]`** | Step selection ±1 along depth axis |
| **Mouse wheel** | Step selection ±1 along depth axis (scroll up = away from camera / same as `]`) |
| **Ctrl + Mouse wheel** | Zoom in/out around selection |
| **Ctrl + `[` / `]`** | Zoom out / in (same as Ctrl+wheel down/up) |
| **LMB** | Select cell |
| **RMB** | Apply active tool |
| **`P` / `E` / `I`** | Place / Erase / Inspect tool (unchanged) |
| **Shift + `1`–`9`** | Palette slot (unchanged) |
| **`?`** | Toggle help overlay |

**Deprecated** (remove from code and README):

| Input | Former action |
|-------|----------------|
| `1` / `2` / `3` | Fixed view switch |
| `F` / `L` | Front / Left view |
| `PageUp` / `PageDown` / `-` / `=` | Alternate slice keys (optional: keep as aliases for `[` / `]` if low cost) |

## Help overlay

- **Toggle:** `?` opens/closes a centered modal panel (Bevy UI).
- **Content:** grouped sections — Selection & navigation, Views & zoom, Tools & palette, Sim controls (link to existing P5 HUD keys).
- **Dismiss:** `?` again, **Esc**, or click outside (Esc reserved for future settings menu; closing help must not open settings).
- **Style:** match factory overlay typography; semi-transparent backdrop; scrollable if content exceeds viewport.

## Data flow

```text
seed_demo_world → random SelectedCell
       ↓
ActiveView + SelectedCell → slice_depth → visible_surface → render
       ↓
camera look-at(selected) + zoom_cells
       ↓
Factory UI inspector ← SelectedCell
```

## Acceptance criteria

- [x] On startup, one cell is selected (random non-empty in demo seed) and visible in the inspector.
- [x] Switching views (Ctrl+arrows) keeps the same world cell selected and centers it in the viewport.
- [x] `T` returns to Top with default orientation without clearing selection.
- [x] Arrow keys move selection on the UV plane; `[`/`]`/wheel move selection along depth; slice clip matches selected depth.
- [x] Ctrl+wheel and Ctrl+`[`/`]` zoom between 8 and 64 cells visible (shorter axis); camera stays centered on selection.
- [x] LMB updates selection only; RMB applies place/erase at picked location per active tool.
- [x] Inspector always shows the selected cell and updates when sim state changes at that coordinate.
- [x] `?` toggles help overlay listing bindings; Esc closes it.
- [x] All six faces render correct visible surfaces (unit tests for surface + UV + adjacency table).
- [x] README **Using the app** section updated; P4/P5 specs cross-link this doc.

## Implementation notes (2026-07-17)

| Decision | Choice |
|----------|--------|
| Camera model | Single `GridCamera` entity; pose + ortho zoom updated each frame from `ActiveView` + `SelectedCell` |
| `InspectedCell` / `ViewSlice` | Removed; `SelectedCell` + `OrthoView::slice_depth` |
| Initial selection | Deterministic hash pick from demo seed positions (no `rand` dependency) |
| LMB drag | Updates selection to cell under cursor while held |
| Wheel without Ctrl | Slice step; sign matches `[` (scroll up decreases depth coordinate) |
| PageUp/PageDown aliases | Dropped |
| Selection highlight | Not implemented (optional nice-to-have) |
| Help toggle | `Shift`+`/` (`?`); Esc or backdrop click dismisses |
